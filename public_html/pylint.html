Pylint

Posted on November 23, 2021 by Now-U-Know. Image Credit:Kowsalya Vijayan

Pylint is a quality checker tool for the python programming language.It checks the error in the source code and enforces code standard.It can also recommend suggestions on adding details on your code.It is named

following a common convention in Python of a "py" prefix, and a nod to the C programming lint program. It follows the style recommended by PEP 8, the Python style guide.Some features of Pylint are listed below. Check pylint.org for complete list

Coding Standard

  • Checking the length of each line

  • checking if variable names are well-formed according to your coding standard

  • checking if imported modules are used

Error detection

  • checking if declared interfaces are truly implemented

  • checking if modules are imported

Refactoring help

  • detects duplicate code

Fully customizable

  • Pylint is highly configurable, customizable, and you can easily write a small plugin to add a personal feature.

Best Practises and Tips for new Programmers

The following guidelines helps to write clear code with the goal of improving readability. Code is read much more often than is written that is why is so important to stick to conventions that will help us or other people to understand more easily the available code.

Operators

Always surround these operators with a single space on either side:

  • Assignment (=)
  • Augmented assignment (+=, -=, etc)
  • Comparison operators (<, >, <=, >=, ==, !=)
  • Logical operators (and, or, not)
  • Membership operators (in, not in)
  • Identity operators (is, not is)

Whitespaces

  • After a comma, semicolon, or colon, we have to use a whitespace but not before them
  • No whitespaces are used immediately inside parentheses, brackets, or braces

Keyword and default arguments

An argument is a value passed to a function when calling it.Python functions accept two types of arguments: (1) positional arguments, and (2) keyword arguments. When calling a function using positional arguments, the arguments must be included in the proper order. On the contrary, the keyword arguments can be provided in an arbitrary order, since these arguments are preceded by an identifier (keyword=value). Function arguments can have default values. In case the argument is not provided when calling a function, the argument takes its default value.

Exceptions

when you write try and except blocks in your code. Do not leave your except bare.Instead ,We should specify which error we want to handle in the except block as

Boolean variables

It is a bad practise to compare a boolean variable to True or False in if statements comparison to True should be just the expression ex. if(is_true)

Import

Imports are written in separated lines, and are grouped in the correct order.
