Testing with AAA pattern for Python

Posted on November 23, 2021 by Now-U-Know. Image Credit:Kowsalya Vijayan
AAA overview

Unit tests are an essential part of software development that independently test a part of the app. Unit testing works in three steps, also known as AAA (Arrange, Act, Assert):

  • It initializes the part of the code that needs to be tested.

  • It applies some stimulus to the system under observation.

  • It records and observes the results.

The advantage of this focus is that it clearly separates the arrangement of the System Under Test and the assertions that are made on it after the action

You might have “clean” code in the main codebase, but disorganisation and inconsistency in the test suite. However when AAA is applied, I’ve found it helps by unifying and clarifying the structure of tests which helps make the test suite much more understandable and manageable.

AAA demonstration using Calculator Program
File structure

  1. Calculator.py : The Calculator class is defined in Calculator.py. It has four functions that implements addition, subtraction, multiplication and division.
  2. Calculator_test.py: The TestCalculator class is defined in test.py. It is a unittest.TestCase class’s subclass. It opens and loads the test data from the test_data.csv file in the class level setup function. It closes the test_data.csv file in the class level teardown function. It also has four test functions to test addition,subtraction ,multiplication and division which test the Calculator class’s related functions
  3. test_data.csv: This is a CSV file that stores test data. One row of text in this file contains six columns separated by a comma.

Unit test library has been imported and unit test case class has been created that extends the unittest.TestCase.

import unittest
class TestApp(unittest.TestCase)

The arrange section is where you setup the objects to be implemented.You bring the system under test to a desired state and configure the dependencies.

For the test preparation we have the class-level setup function which will be executed before all test functions only once, while the instance-level

setup function will be executed before each test function These functions will instantiate the object and will load the data from external file to conduct the test
