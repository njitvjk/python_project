Pytest

You would have read unit tests and AAA in previous webpage you may have learnt Python’s built-in unittest module. unittest is proved to be a base to build your test suite, but it has a few shortcomings. There are a lot of third-party testing frameworks to address issues with unittest, and pytest has proven to be one of the most popular. pytest is a feature-rich,

plugin-based ecosystem for testing your Python code.Some advantages of Pytest are:

  • It is easy and simple to understand

  • It can automatically detect the test cases

  • it can run tests in parallel

  • it can skip tests

PyTest is a testing framework helping you to write simple and scalable test cases for databases, APIs, or UI for python programming language. PyTest is mainly used for writing tests for APIs. It helps to write tests from simple unit tests to complex functional tests.

Pytest installation
Step1:Install
pip install -U pytest
Step2:Test
$ pytest --version

pytest presents the test results differently than unittest. The report shows:

  1. The system state, including which versions of Python, pytest, and any plugins you have installed

  2. The rootdir, or the directory to search under for configuration and tests.

  3. The number of tests the runner discovered.

The output then indicates the status of each test using a syntax similar to unittest:

  • A dot (.) means that the test passed.

  • An F means that the test has failed.

  • An E means that the test raised an unexpected exception.


Pytest Fixtures

Fixtures are used when we want to run some code before every test method. So instead of repeating the same code in every test we define fixtures. A test method can use a Pytest fixture by mentioning the fixture as an input parameter.

A method is marked as a Pytest fixture by marking with

@ pytest.fixture

Coverage with Pytest

Code coverage is the measure of coverage by the test suite. 100% coverage represents that the system has been completely tested

Installation
pip install pytest-cov
Usage
pytest --cov=myproj tests/
Example
pytest --cov --cov-report term --missing

Third Party tool- Coveralls

Coveralls is a web service to help you track your code coverage over time, and ensure that all your new code is fully covered.

You should have a repo hosted in Git and authorize Coveralls to use the repo and its branches
